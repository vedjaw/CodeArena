{# ─── FEATURES (asymmetric grid) ─────────────────────────────────── #}
01 / engine

An assessment surface
that holds up under load.

Six modules · one
contiguous control plane.
/ 001 — IDE

A real editor. Not a textarea pretending.

Multi-language editor with syntax highlighting, custom input, and per-test feedback. Auto-saves drafts every 30s — candidates never lose work to a flaky connection.

/ 002 — SANDBOX

Docker-isolated execution.

Every submission runs in an ephemeral container with time and memory limits. The host is never reachable.

/ 003 — PROCTOR

Anti-cheat by construction.

Tab switches, blur events, copy/paste, right-click, fullscreen exits — all instrumented with audit logs.

/ 004 — TIMING

Deadline-aware sessions.

Server-authoritative timers, auto-submit on expiry, freeze-points for leaderboards in late stages.

/ 005 — LEADERBOARD

Real-time rank surface.

Live scoring, freeze states, tiebreakers, and per-question acceptance graphs. Export anytime.

/ 006 — ANALYTICS

Decisions you can actually defend.

Per-question pass rate, score distribution, violation matrix, time-to-first-AC, and exportable PDF / CSV / Excel reports for every contest.

{# ─── FLOW (numbered editorial steps) ────────────────────────────── #}
02 / flow

Four steps. Zero ceremony.

From draft to
signal in ~6 minutes.
/ STEP 01
Author

Compose the test.

Drop in coding problems, MCQs, or open-ended prompts. Define test cases. Set difficulty. Mix and match — no fixed templates.

/ STEP 02
Distribute

Invite or publish.

Email-gated invitations, password-protected entry, or public listings. Track who started, finished, or abandoned.

/ STEP 03
Monitor

Watch the room.

Live status, violation events, and submission stream. Freeze the leaderboard. Reset a candidate. Announce mid-contest.

/ STEP 04
Decide

Read the signal.

Score distribution, per-question difficulty calibration, integrity flags. Export everything. Make the call.
